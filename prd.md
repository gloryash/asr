## 2. 产品概述

### 2.1 产品背景

**核心洞察**：
- PRD 文档缺乏用户画像和场景细节
- 用户画像需具体鲜活（姓名、背景、偏好）
- 需通过 design thinking 工具细化需求
- 先花时间对齐需求再开发，可大幅减少 AI 交互纠错

**当前痛点**：
1. 用户有 idea 但不知道如何系统化表达
2. 直接让 AI 编程，需求不清导致反复修改
3. 缺乏专业产品经理引导需求梳理

### 2.2 产品定位

**一句话描述**：通过 AI 语音采访，帮助用户将模糊的 idea 转化为详尽的 PRD 文档。

**核心价值**：
- 降低需求表达门槛（语音比文字更自然）
- AI 主动引导（用户不需要知道 PRD 怎么写）
- 实时结构化（边聊边生成文档）

### 2.3 成功指标

| 指标 | 目标值 |
|------|--------|
| 单次采访时长 | 5-10 分钟 |
| PRD 信息完整度 | ≥ 80% 必填项 |
| 用户满意度 | ≥ 4/5 分 |
| AI 编程成功率提升 | 减少 50% 返工 |

## 3. 用户分析

### 3.1 目标用户

**主要用户**：有产品想法但缺乏产品经理经验的开发者/创业者

### 3.2 用户画像

#### 画像 1：小明 - 技术创业者

| 属性 | 描述 |
|------|------|
| 姓名 | 小明 |
| 年龄 | 28 岁 |
| 职业 | 全栈开发者，正在做 side project |
| 技术水平 | 熟练编程，但不懂产品方法论 |
| 痛点 | 脑子里有很多想法，但写出来总是不完整；直接让 AI 写代码，经常需要反复修改 |
| 目标 | 快速把想法变成可执行的需求文档 |
| 使用场景 | 晚上有灵感时，打开网页跟 AI 聊 10 分钟，第二天就有完整 PRD |

#### 画像 2：Lisa - 产品新人

| 属性 | 描述 |
|------|------|
| 姓名 | Lisa |
| 年龄 | 24 岁 |
| 职业 | 刚转行的产品经理 |
| 技术水平 | 会用 Figma，了解基本产品概念 |
| 痛点 | 知道 PRD 该有什么，但不知道怎么问出关键信息 |
| 目标 | 学习专业的需求采访技巧 |
| 使用场景 | 在跟客户沟通前，先用 AI 模拟采访流程 |

### 3.3 用户痛点

1. **不知道该说什么**：面对空白文档不知从何下手
2. **容易遗漏关键信息**：没有系统性框架引导
3. **表达不够具体**：说"做个电商网站"但细节不清
4. **无法自我追问**：缺乏追问和深挖的能力

### 3.4 用户目标

1. 用最自然的方式（说话）表达想法
2. 在 AI 引导下逐步完善需求
3. 获得可直接用于开发的 PRD 文档

## 4. 功能需求

### 4.1 功能概览

```
┌─────────────────────────────────────────────────────────────┐
│                     AI 采访助手                              │
├─────────────────────────────────────────────────────────────┤
│  核心功能                                                    │
│  ├── 语音对话（ASR + TTS）        ← 已有，复用               │
│  ├── 采访引导（状态机 + Prompt）   ← 新增                    │
│  ├── 信息提取（LLM 结构化）        ← 新增                    │
│  └── PRD 生成（模板 + 润色）       ← 新增                    │
├─────────────────────────────────────────────────────────────┤
│  辅助功能                                                    │
│  ├── 进度指示                                               │
│  ├── 中途预览                                               │
│  ├── 手动跳转阶段                                           │
│  ├── PRD 导出                                               │
│  └── 聊天记录导出                                           │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 详细功能说明

#### F1: 语音对话（已有）

| 项目 | 说明 |
|------|------|
| 描述 | 用户语音输入，AI 语音输出 |
| 现状 | 已实现，复用 VoiceChatEngine |
| 改动 | 无需修改 |

#### F2: 采访引导（核心新增）

| 项目 | 说明 |
|------|------|
| 描述 | AI 按阶段主动提问，引导用户表达需求 |
| 采访阶段 | 开场 → 愿景 → 用户 → 功能 → 场景 → 约束 → 优先级 → 总结 |
| 每阶段行为 | 1. 设置阶段专属 systemPrompt<br>2. AI 主动提问<br>3. 追问不清晰的点<br>4. 确认后进入下一阶段 |
| 阶段推进 | 自动（信息收集完整）+ 手动（用户点击跳转）|

**采访阶段详情**：

| 阶段 | 目标 | 核心问题 | 时长 |
|------|------|---------|------|
| opening | 建立关系，说明流程 | "准备好开始了吗？" | 30-60s |
| product_vision | 理解产品是什么 | "你想做什么产品？解决什么问题？" | 60-120s |
| target_users | 明确目标用户 | "谁会用？他们有什么特点？" | 60-120s |
| core_features | 梳理核心功能 | "必须有哪些功能？优先级如何？" | 90-180s |
| user_scenarios | 描述使用场景 | "用户打开产品后会怎么操作？" | 60-120s |
| constraints | 了解限制条件 | "有技术/时间/资源限制吗？" | 45-90s |
| priorities | 确认优先级 | "如果只能做 3 个功能，选哪些？" | 30-60s |
| summary | 总结确认 | "我理解的是...对吗？" | 60-120s |

#### F3: 信息提取（核心新增）

| 项目 | 说明 |
|------|------|
| 描述 | 从用户回答中实时提取结构化信息 |
| 实现方式 | 每轮对话后调用 LLM 提取 JSON |
| 信息分类 | overview / users / features / scenarios / constraints / priorities |
| 触发时机 | 用户说完话后延迟 2 秒（节流）|

#### F4: PRD 生成（核心新增）

| 项目 | 说明 |
|------|------|
| 描述 | 基于收集的信息生成 PRD 文档 |
| 草稿生成 | 实时填充模板，显示在预览面板 |
| 最终生成 | 采访结束后调用 LLM 润色 |
| 输出格式 | Markdown |

#### F5: 进度指示

| 项目 | 说明 |
|------|------|
| 描述 | 显示当前采访阶段和整体进度 |
| UI 元素 | 进度条 + 阶段标签 |

#### F6: 中途预览

| 项目 | 说明 |
|------|------|
| 描述 | 采访过程中实时查看已收集的 PRD 信息 |
| UI 元素 | 右侧滑出面板 |

#### F7: 手动跳转阶段

| 项目 | 说明 |
|------|------|
| 描述 | 允许用户跳过或返回某个采访阶段 |
| UI 元素 | 进度条上的可点击阶段标签 |

#### F8: PRD 导出

| 项目 | 说明 |
|------|------|
| 描述 | 将生成的 PRD 导出为文件 |
| 格式 | Markdown (.md) |

#### F9: 聊天记录导出

| 项目 | 说明 |
|------|------|
| 描述 | 实时导出采访过程中的对话记录 |
| 格式 | Markdown (.md) |
| 内容 | 包含用户输入和 AI 回复的完整对话历史 |
| 触发方式 | 用户手动点击导出按钮 |

### 4.3 功能优先级

| 优先级 | 功能 | 理由 |
|--------|------|------|
| **P0** | 采访引导 | 核心功能 |
| **P0** | 信息提取 | 决定 PRD 质量 |
| **P0** | PRD 生成 | 核心输出 |
| **P1** | 进度指示 | 用户体验 |
| **P1** | 中途预览 | 用户明确要求 |
| **P2** | 手动跳转 | 提升灵活性 |
| **P2** | PRD 导出 | 基础需求 |
| **P2** | 聊天记录导出 | 便于回顾和存档 |

## 5. 使用场景

### 5.1 场景 1：小明的 Side Project

**背景**：小明晚上躺在床上，突然想到一个 AI 写周报的工具。

**流程**：
1. 打开采访页面，点击"开始采访"
2. AI 开场介绍，说明流程
3. AI 主动提问："你想做什么产品？"
4. 小明回答，AI 追问细节
5. 采访过程中，小明点击"预览"查看已收集信息
6. 采访结束，生成完整 PRD
7. 导出 Markdown，用于 AI 编程

### 5.2 场景 2：Lisa 的客户需求采访

**背景**：Lisa 要去跟客户沟通需求，先用 AI 模拟。

**流程**：
1. 打开采访助手，开始模拟
2. AI 按专业流程提问，Lisa 扮演客户
3. 学习 AI 的提问技巧
4. 结束后查看 PRD，了解该收集哪些信息

## 6. 约束与风险

### 6.1 技术约束

| 约束 | 说明 |
|------|------|
| 运行环境 | Web 浏览器（需支持 AudioWorklet） |
| API 依赖 | 阿里云 DashScope（ASR/TTS/LLM） |
| 现有代码 | 复用 VoiceChatEngine，不修改 |

### 6.2 风险与应对

| 风险 | 应对 |
|------|------|
| LLM 提取不准确 | 低温度 + JSON 格式 + 人工确认 |
| 对话不自然 | 优化 Prompt，加入对话示例 |
| 阶段推进不合理 | 支持手动跳转 |
